# Contributing to Mien Termin

Thank you for considering contributing to Mien Termin! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub with:

- A clear title and description
- Steps to reproduce the bug
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Your environment (OS, browser, PHP/Node version)

### Suggesting Features

We love feature requests! Please open an issue with:

- A clear title and description
- Why this feature would be useful
- How it should work
- Any implementation ideas

### Pull Requests

1. Fork the repository
2. Create a new branch from `main`
3. Make your changes
4. Write or update tests as needed
5. Ensure all tests pass
6. Update documentation
7. Submit a pull request

### Coding Standards

#### Backend (PHP/Laravel)

- Follow PSR-12 coding standards
- Use meaningful variable and function names
- Add PHPDoc comments for classes and methods
- Keep functions small and focused

#### Frontend (TypeScript/React)

- Use TypeScript for all new code
- Follow the existing code structure
- Use functional components with hooks
- Add proper type definitions
- Keep components small and reusable

### Commit Messages

Write clear commit messages:

- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Start with a capital letter
- Keep the first line under 50 characters
- Add detailed description if needed

Example:

```
Add patient review filtering

- Add filter by rating
- Add filter by approval status
- Add search by patient name
```

## Code of Conduct

Please be respectful and considerate of others. We want to maintain a welcoming community.

## Questions?

Feel free to open an issue with the "question" label.
